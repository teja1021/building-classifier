ADDING YOUR OWN MODEL
================================================================================

1. TRAIN YOUR MODEL
   Use PyTorch with ResNet18 or custom architecture
   Ensure:
     - Input size: 224×224
     - Classes: 17 (match labels.json order)
     - Preprocessing: ImageNet normalization
       mean=[0.485, 0.456, 0.406]
       std=[0.229, 0.224, 0.225]

2. SAVE MODEL
   import torch
   torch.save(model, "backend/app/models/resnet18_best.pt")

3. RESTART BACKEND
   Backend will auto-load on startup

4. VERIFY
   - Call /predict endpoint
   - Check backend logs for "Model loaded successfully"

See backend/app/models/README.txt for detailed instructions.

CONFIGURATION & CUSTOMIZATION

CHANGE API BASE URL (Frontend):
  Edit: frontend/src/api.js
  const API_BASE = 'http://your-server:8000';

CHANGE INPUT IMAGE SIZE:
  Edit: backend/app/inference.py
  INPUT_SIZE = 224  # Change this value
  Retrain model with new size!

  Edit: backend/app/inference.py
  IMAGENET_MEAN = [0.485, 0.456, 0.406]

  Edit: backend/app/main.py
  allow_origins=["https://your-domain.com"]  # Not "*"


TROUBLESHOOTING
================================================================================

Backend not starting:
  $ lsof -i :8000  # Check if port in use
  $ python -m venv .venv  # Recreate venv
  $ pip install -r requirements.txt  # Reinstall deps

Frontend can't reach backend:
  - Is backend running? curl http://localhost:8000/ping
  - Check browser console for CORS errors
  - Update API_BASE in frontend/src/api.js

Model not loading:
  - Check: backend/app/models/resnet18_best.pt exists
  - Verify PyTorch version matches
  - Check: class count = len(labels.json)
  - View backend logs

Slow predictions:
  - GPU available? torch.cuda.is_available()
  - Try ONNX export for faster inference
  - Use smaller model (MobileNet)
